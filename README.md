# Два Telegram-бота под одним запуском

В репозитории два независимых бота, которые поднимаются одним процессом-супервизором:

1. **Прокси-бот** (`main.py` + пакет `bot/`) — продажа прокси, баланс, промокоды,
   рефералка, блокировки. Свой SQLite.
2. **Сауша-бот** (`sausha-bot-main/`) — анонимные комментарии к постам канала (aiogram)
   и анонимные сообщения в канал + админка (python-telegram-bot). Данные в JSON.

Боты **изолированы**: каждый запускается отдельным подпроцессом в своей папке, поэтому
пакет `bot/` прокси-бота и файл `bot.py` сауши не конфликтуют по имени `bot`.

## Запуск локально

```powershell
pip install -r requirements.txt
python run_all.py
```

Отдельно можно запускать и по одному:

```powershell
python main.py                      # только прокси-бот
cd sausha-bot-main; python main.py  # только сауша-бот
```

## Деплой на Render

- **Тип сервиса:** Web Service (супервизор держит HTTP на `$PORT` для healthcheck).
- **Build command:** `pip install -r requirements.txt`
- **Start command:** `python run_all.py`
- **Runtime:** Python 3.11 (см. `runtime.txt`).

Супервизор `run_all.py`:
- открывает `$PORT` (Render считает сервис живым);
- запускает оба бота подпроцессами и **перезапускает** упавший (backoff 5→60 сек);
- подхватывает корневой `.env`, если он есть (на Render переменные задаются в дашборде,
  и они имеют приоритет над `.env`).

## Переменные окружения

### Прокси-бот
| Переменная | Обязательна | Описание |
|---|---|---|
| `BOT_TOKEN` | да | токен прокси-бота |
| `ADMIN_IDS` | да | ID админов через запятую |
| `PROXY6_API_KEY` | нет | ключ Proxy6 (без него покупка прокси не работает) |
| `CRYPTO_PAY_TOKEN` | нет | токен CryptoBot для оплаты |
| `DB_PATH` | нет | путь к SQLite (по умолчанию `data/bot.sqlite3`) |
| `START_IMAGE_PATH`, `PROFILE_IMAGE_PATH`, `TOP_UP_IMAGE_PATH` | нет | картинки экранов |

### Сауша-бот
| Переменная | Обязательна | Описание |
|---|---|---|
| `BOT1_TOKEN` | да | токен бота анонимных комментариев |
| `BOT2_TOKEN` | да | токен бота анонимных сообщений |
| `BOT1_CHANNEL_ID`, `BOT1_DISCUSSION_CHAT_ID` | нет | ID канала и чата обсуждения |
| `BOT2_CHANNEL_ID`, `BOT2_USERNAME`, `BOT2_CHAT_INVITE` | нет | канал/имя/инвайт для бота 2 |
| `ADMIN_ID` | нет | ID админа сауша-бота (у него своя админка) |
| `RENDER_URL` | нет | URL сервиса для self-ping keep-alive |
| `SAUSHA_HTTP_PORT` | нет | внутренний порт keep-alive сауши (по умолчанию 10000; под супервизором можно не трогать) |

> `ADMIN_IDS` (прокси) и `ADMIN_ID` (сауша) — **разные** переменные: у каждого бота своя админка.

## Блокировки

В админке **каждого** бота есть управление блокировками. Вводить можно **числовой ID**
или **@username** (username находится только у тех, кто уже писал этому боту).

- **Прокси-бот:** `/admin` → «🚫 Блокировки» → Заблокировать / Разблокировать.
  Список блокировок хранится в его SQLite (`blocked_users`), отдельно от сауши.
- **Сауша-бот:** в админке бота 1 и бота 2 есть кнопки «🚫 Заблокировать /
  ✅ Разблокировать / 📋 Заблокированные». Блок-лист **общий** для обоих внутренних
  ботов сауши (`sausha-bot-main/blocked_users.json`).

Администраторов заблокировать нельзя. Заблокированные игнорируются на входе.
